Install Cipher

Cipher can be installed on any platform with a single command. Choose your preferred method below.

Recommended: Install Script

The install script auto-detects your platform and installs the latest version.

terminal
# macOS / Linux
$ curl -fsSL https://cipher.elevenpct.com/install.sh | bash

# Windows (PowerShell)
PS> irm https://cipher.elevenpct.com/install.ps1 | iex
Tip: The install script adds Cipher to your PATH automatically. No restart required.

Alternative: npm (Future)

When available, install via npm:

npm install -g @cipher/cli

Verify Installation

cipher --version

You should see v0.3.0 or later.

Requirements

  • OS: Windows 10+, macOS 12+, or Linux (Ubuntu 20.04+, Debian 11+, etc.)
  • Node.js: 18+ (only for npm install method)
  • RAM: 4GB minimum (8GB recommended for local models)
  • Internet: Required for cloud providers; optional for local models

Quick Start

Get up and running with Cipher in under 2 minutes.

Step 1: Navigate to your project

cd my-project

Step 2: Start Cipher

cipher

Step 3: Choose your provider

On first run, Cipher will guide you through provider setup:

cipher setup
> Choose an AI provider:
1. Ollama (free, local)
2. OpenAI (API key required)
3. Anthropic (API key required)
4. GitHub Copilot (login required)
5. Configure manually

> Select [1-5]: 1

Step 4: Start coding

> Create a REST API with Express.js for a todo app

Cipher will analyze your project, understand the context, and generate the code with diffs you can review.

Tip: Use /help to see all available slash commands at any time.

First Run

What happens when you run Cipher for the first time.

Initialization

On first run, Cipher creates a .cipher/ directory in your project root:

.cipher/
  config.json      # Your provider and model settings
  sessions/        # Session history
  cache/           # LSP and model cache
  logs/            # Debug logs

Provider Detection

Cipher automatically scans for available providers:

  • Checks if Ollama is running on localhost:11434
  • Checks for OLLAMA_BASE_URL environment variable
  • Checks for API keys in environment (OPENAI_API_KEY, ANTHROPIC_API_KEY, etc.)
  • Checks for GitHub Copilot token

Interactive Setup Wizard

If no provider is detected, Cipher launches an interactive wizard to help you configure your first provider. The wizard supports all 75+ providers.

Sandbox Boundaries

Cipher establishes sandbox boundaries at startup:

  • Only files within your project directory can be read or modified
  • System files, user home directory, and other projects are off-limits
  • Command execution requires explicit confirmation

Configuration File

Cipher uses a config.json file in your project's .cipher/ directory.

Example Config

.cipher/config.json
{
  "provider": "ollama",
  "model": "qwen2.5-coder:32b",
  "temperature": 0.7,
  "max_tokens": 4096,
  "system_prompt": "You are a senior developer...",
  "sandbox": true,
  "auto_apply": false,
  "context_files": ["**/*.ts", "**/*.js"],
  "exclude": ["node_modules", "dist"]
}

Custom System Prompt

Customize Cipher's behavior with a custom system prompt.

Setting a System Prompt

In your .cipher/config.json:

{
  "system_prompt": "You are a senior TypeScript developer. Follow these rules:\n- Always use strict null checks\n- Prefer interfaces over types\n- Write comprehensive JSDoc comments\n- Use async/await over promises"
}

Per-Project Prompts

Each project can have its own system prompt. This is useful for enforcing project-specific conventions.

Dynamic Variables

System prompts support dynamic variables:

  • {{project_name}} - Current project directory name
  • {{language}} - Detected primary language
  • {{framework}} - Detected framework (React, Express, etc.)
  • {{date}} - Current date

Project Isolation

How Cipher keeps your project safe.

Sandbox Model

Cipher operates within a strict sandbox:

  • Read access: Only files within your project directory
  • Write access: Only files within your project directory
  • Command execution: Requires explicit user confirmation
  • Network access: Only for AI provider API calls

Excluded Paths

By default, Cipher ignores these directories:

"exclude": [
  "node_modules",
  ".git",
  "dist",
  "build",
  ".next",
  "coverage",
  ".cipher"
]

Override Sandbox

For trusted projects, you can disable sandboxing:

{
  "sandbox": false
}
Warning: Disabling sandbox allows Cipher to read/write files outside your project directory. Only do this if you fully trust the AI provider and your configuration.

  1. Open AWS Console
  2. Navigate to Amazon Bedrock

Common Issues

Solutions to frequently encountered problems.

"Provider not found"

Ensure your provider is correctly configured. Run cipher list to see available providers.

"Model not available"

The specified model may not be supported by your provider. Run cipher models to list compatible models.

"Connection refused" (Ollama)

Make sure Ollama is running:

ollama serve

"Rate limit exceeded"

Your API key has hit rate limits. Wait a few minutes or switch to a different provider.

"Permission denied" (macOS/Linux)

Make the install script executable:

chmod +x install.sh
./install.sh

❓ FAQ

Frequently Asked Questions

Everything you need to know about Cipher.

What is Cipher?

▾

Cipher is an autonomous coding agent that runs in your terminal. It connects to 75+ AI providers and can read, write, and edit code in your project directory. Think of it as an AI pair programmer that lives in your command line, works with any editor, and gives you complete control over which AI model to use.

Is it free?

▾

Yes! Cipher itself is completely free and open source under the MIT license. However, some AI providers require API keys with associated costs. You can use Cipher for free with local providers like Ollama, LM Studio, or llama.cpp, which run models on your own hardware at no cost.

What AI providers does it support?

▾

Cipher supports 75+ providers across four categories:

Free/Local: Ollama, LM Studio, vLLM, llama.cpp, Jan.ai

Cloud Free Tiers: DeepSeek, Groq, Google Gemini, HuggingFace, Fireworks AI, Azure AI, Cohere

Premium: OpenAI, Anthropic, Mistral, xAI/Grok, Together AI, OpenRouter, Perplexity, Cerebras, DeepInfra, Sambanova

Enterprise: AWS Bedrock, Google Vertex AI, Azure OpenAI, IBM Watsonx, Oracle Cloud AI

Does it work offline?

▾

Yes, when using local providers like Ollama, LM Studio, vLLM, or llama.cpp. These providers run AI models directly on your hardware, so no internet connection is needed. Cloud providers obviously require an internet connection for API calls.

How is it different from Cursor or GitHub Copilot?

▾

Unlike Cursor (which is a standalone editor) or Copilot (which is tied to VS Code), Cipher is editor-agnostic. It runs in your terminal and works with any editor — Vim, VS Code, Sublime, IntelliJ, or even no editor at all. Additionally, Cipher doesn't lock you into a single AI provider; you can switch between 75+ providers at any time.

Is my code sent to AI providers?

▾

When using cloud providers, yes — your code context is sent to the provider's API for processing. However, Cipher is sandboxed and only sends files within your project directory. When using local providers (Ollama, LM Studio, etc.), your code never leaves your machine. For maximum privacy, use a local provider.

Can I use my own API keys?

▾

Absolutely. Cipher is designed to work with your own API keys. Simply set the appropriate environment variable (e.g., OPENAI_API_KEY, ANTHROPIC_API_KEY) and Cipher will use it. You maintain full control over your API usage and billing.

How do I install it?

▾

macOS/Linux:

curl -fsSL https://cipher.elevenpct.com/install.sh | bash

Windows (PowerShell):

irm https://cipher.elevenpct.com/install.ps1 | iex

npm (future):

npm install -g @cipher/cli

Does it work on Mac/Linux?

▾

Yes! Cipher is cross-platform and works on macOS (12+), Linux (Ubuntu 20.04+, Debian 11+, and most major distributions), and Windows 10+. The terminal interface is consistent across all platforms.

Can I customize the AI behavior?

▾

Yes. You can customize Cipher's behavior through:

  • System prompts — Set custom instructions in your config
  • Temperature — Control creativity (0.0 to 2.0)
  • Max tokens — Limit response length
  • Context files — Choose which files are loaded as context
  • Sandbox — Enable/disable file access restrictions
  • Auto-apply — Toggle automatic diff application

How do I update Cipher?

▾

Run the update command:

cipher update

Or re-run the install script, which will detect the existing installation and upgrade it.

Is it open source?

▾

Yes! Cipher is 100% open source under the MIT license. You can view the source code, contribute, fork, or modify it however you like. Find it on GitHub at github.com/cipher-agent/cipher.
