Install

Cipher requires Python 3.10 or higher. The installer handles everything else.

Windows

irm https://cipher.elevenpct.com/install.ps1 | iex

macOS / Linux

curl -fsSL https://cipher.elevenpct.com/install.sh | bash
Tip: Make sure Python is installed first. On macOS: brew install python3. On Linux: sudo apt install python3 python3-pip.

Quick Start

# Navigate to your project
cd my-project

# Run setup (interactive)
cip --setup

# Start coding
cip

First Run

On first run, Cipher will guide you through selecting an AI provider and model. You will see:

  1. A list of providers grouped by Free / API Key
  2. Click a provider to select it
  3. If the provider has multiple models, choose one
  4. If an API key is needed, you will be prompted to enter it

Your selection is saved to ~/.cipher/config.json and used for future sessions.

Config File

Cipher saves settings to ~/.cipher/config.json:

{
  "provider": "ollama",
  "model": "ollama/qwen3:14b",
  "show_plan": true,
  "show_code": true,
  "show_diff": true,
  "show_tool_exec": true,
  "compact_mode": false,
  "auto_confirm": false,
  "expand_explanations": false
}

You can edit this file directly or use the settings panel (Ctrl+S) in the TUI.

Commands

# Start with interactive setup
cip --setup

# Specify provider and model
cip --provider ollama --model ollama/qwen3:14b

# Specify working directory
cip --dir /path/to/project

# Use custom API key
cip --api-key your-key-here

System Prompt

Cipher sends a system prompt to the AI that instructs it to use XML-style action tags: <plan>, <write>, <run>, <read>, <ls>. You can modify this in cipher/app.py if needed.

Project Isolation

Cipher validates all file paths against the working directory. It will reject any attempt to access files outside the project root, including ../ escapes and absolute paths.

Common Issues

Command not found: cip

Make sure Python's Scripts directory is in your PATH. On Windows: %LOCALAPPDATA%\Programs\Python\Python312\Scripts

Blank screen on startup

This was fixed in v0.0.3. If you still see it, update: pip install --upgrade cipher-agent

Ollama connection refused

Make sure Ollama is running: ollama serve
